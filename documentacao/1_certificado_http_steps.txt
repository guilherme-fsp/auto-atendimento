Tirando certificado https:

1. Instalar o Certbot e plugin Nginx (Oracle Linux 7.9)

sudo yum install -y epel-release
sudo yum install -y certbot python2-certbot-nginx

2. Apontar na cloudFare o ip do meu programa

3. Listar o firewall

sudo firewall-cmd --list-all

4. Adicionar excessao

sudo firewall-cmd --permanent --add-service=https

5. Reiniciar meu firewall

sudo firewall-cmd --reload

6. Verificar status do meu nginx

sudo systemctl status nginx

7. Rodei o certbot para emitir o certificado e permitir a comunicação

sudo certbot --nginx -d integravoce.com -d www.integravoce.com

8. Deu erro porque meu nginx file nao estava configurado corretamente. Entao usei essa modificação

# Redireciona HTTP para HTTPS
server {
    listen 80;
    server_name integravoce.com www.integravoce.com;

    return 301 https://$host$request_uri;
}

# Bloco HTTPS com proxy e SSL
server {
    listen 443 ssl;
    server_name integravoce.com www.integravoce.com;

    ssl_certificate /etc/letsencrypt/live/integravoce.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/integravoce.com/privkey.pem;

    error_log /var/log/nginx/whatsbot_error.log debug;
    access_log /var/log/nginx/whatsbot_access.log;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    location /static/ {
        alias /home/opc/auto-atendimento/static/;
        autoindex on;
        try_files $uri $uri/ =404;
    }

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

9. Verifiquei erros

sudo nginx -t (comando)
nginx: the configuration file /etc/nginx/nginx.conf syntax is ok (resposta)
nginx: configuration file /etc/nginx/nginx.conf test is successful (resposta)

10.Reiniciei meu servidor nginx

sudo systemctl reload nginx

11. Funcionou.
